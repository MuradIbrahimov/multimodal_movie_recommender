# Multimodal Movie Recommender

A neural collaborative filtering project that predicts MovieLens ratings and compares:

| Model | Description |
|---|---|
| Baseline NCF | Learns user and movie latent embeddings from rating history only. |
| Multimodal NCF | Warm-starts from the baseline and adds a residual branch for genres, TMDB metadata, and poster embeddings. |

The current workflow has one dataset construction path: `dataset/merge.ipynb`. The previous standalone dataset build script is no longer needed.

## Repository Structure

```text
movie_recommender.ipynb              Main training, evaluation, recommendations, and visual diagnostics
extract_poster_embeddings.py         Downloads posters and extracts MobileNetV2 embeddings
dataset/
  merge.ipynb                        Unified processed-dataset construction notebook
  MultimodalMovieDataset_v2/
    README.md                        Dataset audit generated by merge.ipynb
saved_models/                        Trained Keras models
```

Large raw datasets, processed CSVs, poster caches, `.npy` embeddings, and saved model weights are ignored by git.

## Data Pipeline

1. Download MovieLens 25M into `dataset/ML25M/`.
2. Put `TMDB_all_movies.csv` into `dataset/TMDB/`.
3. Put `MovieGenre.csv` into `dataset/MovieGenreFromItsPoster/`.
4. Run `dataset/merge.ipynb` from top to bottom.
5. Run poster embedding extraction:

```bash
python extract_poster_embeddings.py --data-dir dataset/MultimodalMovieDataset_v2
```

`merge.ipynb` writes:

| File | Purpose |
|---|---|
| `final_movie_features_clean.csv` | One row per movie with merged metadata and poster URLs. |
| `final_ratings_clean.csv` | MovieLens ratings with stable `user_idx` and `movie_idx`. |
| `model_sample_500k.csv` | 500k sampled ratings joined with movie features for fast notebook runs. |
| `movie2idx.csv`, `user2idx.csv` | Stable index mappings used by the models. |
| `audit_report.csv` | Coverage checks for posters, genres, metadata, users, movies, and sample rows. |

The optional huge `model_df_clean.csv` is skipped by default. This avoids duplicating all ratings plus all movie metadata unless you explicitly set `WRITE_FULL_MODEL_DF = True`.

## Training Notebook

Open:

```bash
jupyter lab movie_recommender.ipynb
```

Defaults:

| Setting | Meaning |
|---|---|
| `USE_SAMPLE = True` | Train on `model_sample_500k.csv`. |
| `USE_POSTER_SVD = True` | Reduce MobileNetV2 poster embeddings before training. |
| `EPOCHS_BASELINE = 20` | Maximum baseline epochs with early stopping. |
| `EPOCHS_MULTI = 20` | Maximum multimodal epochs with early stopping. |

If `poster_embeddings.npy` is missing, the notebook now uses zero poster features and still runs. That mode is useful for checking the metadata and collaborative branches, but run `extract_poster_embeddings.py` before presenting the full multimodal poster result.

## Model Evaluation and Visuals

`movie_recommender.ipynb` now includes:

- RMSE and MAE comparison bars, including improvement percentage.
- Training and validation loss/MAE curves.
- Prediction error histograms for baseline vs multimodal.
- Actual vs predicted scatter plots.
- MAE by true rating bucket.
- Prediction-delta distribution showing how much the multimodal branch changes the baseline.
- Segment RMSE by poster availability, movie activity, and user activity.
- Permutation feature importance for metadata features, poster visual embeddings, and the poster availability flag.
- Side-by-side top-N recommendation comparison with overlap and score differences.

The recommendation demo now uses a real active training user and only ranks movies whose embeddings were trained, which avoids recommending random untrained sample-only candidates.
